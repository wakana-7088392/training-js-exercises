# ch16/ex11

## 複数のTCPクライアント (net.Socket) でHTTPリクエストを送信せず同時に接続を維持した際、何接続で接続が確立できなくなるか確認し、確立できなかった理由を書きなさい。

- 確認方法
    - index.jsに検証用コードを追加
    - console.logで接続数を確認する
- 結果
    - 16275個接続したのちエラーが出た
    - 9000接続してリソースモニターをみたらTCP接続に大量のnode.exeがあった
- 理由
    - 利用可能なポートの数が限界に達したから
        - netsh int ipv4 show dynamicport txpでエフェメラルポートの数を調べたところ。ポート数は16384だった。
        - 他のもの（chrome等）も開いているためその分ポートが使われていることを考えると16275で残っていたポート数を全て使い切ったのではないだろうか。
