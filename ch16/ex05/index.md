# ch16/ex05
## 標準入力、標準出力、標準エラー出力、リダイレクト、パイプという単語について調べなさい

- 標準入力
    - コンピュータ上で実行されているプログラムが特に何も指定されていない場合に標準的に利用するデータ入力元
    - コマンド/アプリ側で入力元を明示することなく使用できる入力
    - ファイルの入力の話
    - システム上ではstdinの略号で表されることが多い
- 標準出力
    - コンピュータ上で実行されるプログラムが特に何も指定されていない場合に標準的に利用するデータ出力先
    - コマンド/アプリ側で出力先を明示することなく使用できる出力
    - ファイルの出力の話
    - システム上ではstdoutの略号で表されることが多い
- 標準エラー出力
    - コンピュータ上で実行されるプログラムが特に何も指定されていない場合に標準的に利用するエラー出力先
    - システム上ではstderrの略号で表されることが多い
    - エラーが発生した時にエラーコードやメッセージを出力する先のことを指す
    - 多くのシステムではディスプレイへの文字表示が標準エラー出力先に設定されているが、操作や設定により指定ファイルへの書き込みなどに変更することもできる。
- リダイレクト
    - 標準入力元や標準出力先を変更すること
    - コマンドの記号は「>」「<」「>>」？があげられる
    - 出力を画面ではなくファイルに保存することなどが可能
- パイプ
    - コマンドの実行結果を直接別のコマンドの入力として渡すこと
    - 実行結果を連続して他のコマンドで処理できる
    - 前のコマンドの標準出力を、次のコマンドの標準入力として使える

- ls | grep 'ch0' >> a.txt というコマンドをtraining-js-execises上で実行すると、training-js-execises内のch0○のフォルダ名の羅列がa.txtというファイルに入力され、保存される。


## 以下のコードを cat.mjs というファイルに保存し、後述する実験の結果を予測し、実際に実験しなさい

- 実験内容
    - file は適当なファイルとし invalid-file は存在しないファイルとしなさい
    - リダイレクトとパイプがからむコマンドは「stdin is not a tty」が出るため「node.exe」を適宜つけている

- 予測と結果
    - node cat.mjs
        - 予想
            - コマンドに入力した文字を出力する（aと入力したらaが出力される）
        - 結果
            - コマンドに入力した文字を出力する（aと入力してエンターを押下したらaが出力される）
    - echo FOO | node cat.mjs
        - 予想
            - FOOが出力される
        - 結果
            - FOOが出力される
    - node cat.mjs > output.txt
        - 予想
            - 入力した内容がoutput.txtに反映される
        - 結果
            - 入力した内容がoutput.txtに反映される（a b c d と入力したらoutput.txtに反映されていた）
    - node cat.mjs file.txt
        - 予想
            - file.txtにある文字列が出力される
        - 結果
            - file.txtにある文字列「テスト用テキスト」が画面に出力される
    - node cat.mjs file.txt > output.txt
        - 予想
            - file.txtにある文字列がoutput.txtに反映される
        - 結果
            - file.txtにある文字列「テスト用テキスト」がoutput.txtに反映されるためファイルを確認すると以前テストした結果反映されていた「a b c d」から「テスト用テキスト」に変わっている
    - node cat.mjs invalid.txt > output.txt
        - 予想
            - output.txtが空欄になる（文字列が消える）
        - 結果
            - invalid.exeがないためエラーが画面に出力される
    - node cat.mjs invalid-file 2> error.txt
        - 予想
            - エラー内容がerrpr.txtに反映される
        - 結果
            - エラー内容がerror.txtに反映される
