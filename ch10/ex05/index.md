# 各モジュールで、エクスポート元の関数・クラスをエディタのリファクタ機能で名前変更した際、インポート側で名前変更がどう追随されるのかを確認する

## 10.3

- 関数
  - 対象関数：module.cjsのgreet関数
  - 変更内容：greet → gre
  - 実施結果：関数を呼び出している箇所が「gre」に変化している。→　変化しないらしい
- クラス
  - 対象クラス：module.cjsのHumanクラス
  - 変更内容：Human → Ningen
  - 実施結果：インポート側では何も変化しない。

## 10.4

### デフォルトのエクスポート

- 関数
  - 対象関数：module_tanuki.jsのgreetTanuki関数
  - 変更内容：greetTanuki → gre
  - 実施結果：インポート側であるindex.jsでも「gre」に変化していた。
- クラス
  - 対象クラス：module_dog.jsのDogクラス
  - 変更内容：Dog → Inu
  - 実施結果：インポート側であるmodule.jsでも「Inu」に変化していた。

### 名前変更を伴うインポート

- 関数
  - 対象関数：module_dog.jsのgreet関数(module.jsにて名前変更を伴うインポートをしている)
  - 変更内容：greet → gre
  - 実施結果：インポート側であるmodule.jsでも「gre」に変化していた。
- クラス
  - 対象クラス：module_tanuki.jsのTanukiクラス(index.jsにて名前変更を伴うインポートをしている)
  - 変更内容：Tanuki → Ta
  - 実施結果：インポート側であるindex.jsでも「Ta」に変化していた。

### 再エクスポート

- 関数
  - 対象関数：module_tanuki.jsのeat関数(module.jsにて再エクスポートしている)
  - 変更内容：eat → e
  - 実施結果：再エクスポート時、変更後の「e」が変更前の「eat」に名前変更される形(export { e as eat } ...)に変化していた。  
    また、再エクスポートしたものをindex.jsでインポートしているが、インポート側で該当の関数を呼び出している箇所は変更後の「e」に変化していたため、そのままにしておくとエラーになる。
- クラス
  - 対象クラス：module_cat.jsのCatクラス(module.jsにて再エクスポートしている)
  - 変更内容：Cat → Neko
  - 実施結果：再エクスポート時、変更後の「Neko」が変更前の「Cat」に名前変更される形(export { Neko as Cat } ...)に変化していた。  
    また、再エクスポートしたものをindex.jsでインポートしているが、インポート側で該当の関数を呼び出している箇所は「Cat」のままであり、処理上問題はなかった。
